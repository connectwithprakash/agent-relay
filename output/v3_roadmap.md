# Agent Relay v3 Roadmap

Generated by 3-agent strategy team (product_strategist, tech_architect, dx_engineer) communicating via Agent Relay.

---

## Vision

Agent Relay is the **"SQLite of agent communication"** - the simplest, most developer-friendly way for AI agents to coordinate. v3 makes the jump from functional MVP to the tool developers tweet about.

## Killer Differentiators (vs A2A, MCP, COGENT, AgentWorkforce)

1. **Spectator/Observer Mode** - Watch agent conversations without participating. No competitor has this. Enables monitoring, logging agents, human oversight, and audit trails.
2. **Turn Timeout + Auto-Advance** - Prevents relay deadlocks. Configurable per-relay, with auto-skip.
3. **Typed Message Protocol** - Structured message types (task_request, task_result, error, handoff) with validation.
4. **Agent Presence + Heartbeats** - Know which agents are connected, detect crashes.
5. **Flexible Turn Policies** - Beyond round-robin: free-form, priority-based, group channels.

## Milestones

### v3.0-alpha: "The Demo" (3 weeks)
- Spectator mode (read-only WS/SSE endpoint)
- CLI tooling (`agent-relay quickstart`, `agent-relay watch`)
- Python SDK with async iterator (`for msg in relay.listen()`)
- Typed message protocol (additive, non-breaking)
- **Goal**: The 5-minute tweet moment. `pip install agent-relay && agent-relay quickstart` shows agents talking in 60 seconds.

### v3.0: "Production-Ready" (6 weeks)
- Agent heartbeats + presence detection
- Turn timeouts with auto-advance
- MCP bridge class for seamless integration
- All P0 security fixes merged
- **Goal**: Safe for production multi-agent deployments.

### v3.1: "Scale" (10 weeks)
- Flexible turn policies (free-form, priority, groups)
- `create-agent-relay` templates (simple-relay, code-review, mcp-bridge)
- Auto-generated API docs from OpenAPI
- Support for 10+ agents efficiently
- **Goal**: Enterprise-grade multi-agent orchestration.

## Technical Feasibility (per tech_architect)

| Feature | Difficulty | Architecture Impact |
|---------|-----------|-------------------|
| Spectator mode | EASY | New WS/SSE endpoint, zero turn logic changes |
| Typed messages | EASY | Additive enum on existing type field |
| Heartbeats | MEDIUM | Background asyncio task for timeout detection |
| Turn timeouts | MEDIUM | New model columns + scheduler |
| Flexible turns | HARD | Replaces core turn protocol, needs careful design |

## DX Priorities (per dx_engineer)

1. **60-second quickstart** - `agent-relay quickstart` scaffolds a working 3-agent relay locally
2. **3-line SDK join** - `client = AgentRelayClient(url); relay = client.join(id); for msg in relay.listen(): ...`
3. **`agent-relay watch`** - Terminal UI showing live agent conversation (the demo command)
4. **Templates** - `create-agent-relay` with options: simple-relay, code-review, mcp-bridge
5. **Docs-first** - Getting Started, SDK Reference, MCP Integration Guide, Examples Gallery
